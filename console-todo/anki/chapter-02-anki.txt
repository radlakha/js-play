<span class="concept-label">Concept</span><div class="q">What does "event-driven programming" mean in the context of <code>process.stdin</code>, and how does it differ from sequential code?</div>	<div class="concept-answer"><b>Event-Driven vs Sequential</b><br><br>In <b>sequential</b> code, each line runs one after another &mdash; the program controls the flow.<br><br>In <b>event-driven</b> code, the program registers a listener and waits. The runtime calls your function when an event occurs:<br><br><pre>process.stdin.on("data", callback);</pre><br>The callback does not run immediately &mdash; it fires <b>whenever the user presses Enter</b>. The program is idle in between.<br><br>This pattern is fundamental to Node.js: HTTP servers, file watchers, WebSockets, and UI frameworks all work this way.</div>	ch02 events concept
